## §1 はじめに

本マニュアルは、KKBD-USB ファームウェアをゼロからビルドして Raspberry Pi Pico に書き込むまでの手順を説明します。

KKBD-USB は USB キーボードの入力を UART 信号に変換して SBC（シングルボードコンピューター）へ送信するファームウェアです。ターゲット MCU は **Raspberry Pi Pico（RP2040）** であり、ビルドには **Raspberry Pi Pico SDK** と **ARM クロスコンパイラ** が必要です。

本書の位置付けを以下に示します。

```
01_ハードウェア構成.md  ← 必要部品・システム構成・Pico ピンレイアウト
02_組み立て手順.md      ← 配線手順・ジャンパー設定・配線図
03_ビルドと書き込み.md  ← 本書（ビルド環境構築〜Pico への書き込み）
04_使い方.md            ← 起動・操作・ジャンパー設定・制限事項
05_OTGと電源.md         ← USB OTG ケーブルと外部電源の選定
06_キーボード対応状況.md ← 対応キーボード・配列・対応キー一覧
07_トラブルシューティング.md ← 各種トラブル症状・原因・対処
```

## §2 必要なソフトウェア

### 2.1 macOS

[Homebrew](https://brew.sh/) を使用してインストールします。

```sh
# Homebrew 未インストールの場合
/bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

# ビルドツール一式
brew install cmake ninja

# ARM クロスコンパイラ（GCC Embedded）
brew install --cask gcc-arm-embedded

# picotool（任意: .uf2 情報確認や書き込みに使用）
brew install picotool
```

インストール後、バージョンを確認します。

```sh
cmake --version          # CMake 3.13 以上
arm-none-eabi-gcc --version  # 10 以上
```

### 2.2 Linux（Ubuntu / Debian 系）

```sh
sudo apt update
sudo apt install -y cmake ninja-build gcc-arm-none-eabi git
```

> **注意**: Ubuntu 20.04 のパッケージリポジトリに含まれる cmake が古い場合（3.13 未満）は、[Kitware の APT リポジトリ](https://apt.kitware.com/)から最新版をインストールしてください。

バージョン確認:

```sh
cmake --version
arm-none-eabi-gcc --version
```

### 2.3 Windows

以下の 2 通りの方法があります。

#### 方法 A: WSL2 + Linux 手順（推奨）

WSL2（Windows Subsystem for Linux 2）上の Ubuntu 環境で §2.2 の Linux 手順をそのまま実行できます。WSL2 のセットアップは [Microsoft 公式ドキュメント](https://learn.microsoft.com/ja-jp/windows/wsl/install) を参照してください。

#### 方法 B: Pico SDK 公式インストーラ

[Raspberry Pi 公式のインストーラ](https://github.com/raspberrypi/pico-setup-windows) を使用すると、Pico SDK・ARM ツールチェーン・CMake・Visual Studio Code 拡張機能をまとめてセットアップできます。

## §3 Pico SDK のセットアップ

### 3.1 SDK のクローン

**推奨場所**: プロジェクトリポジトリの**外側**（例: ホームディレクトリ `~/pico-sdk`）。

> **注意**: Pico SDK を KKBD-USB リポジトリの内部に clone しないでください。未追跡ファイルとして混入し、誤コミットの原因になります（`.gitignore` で一定は保護していますが、別ディレクトリに置く運用が推奨です）。

```sh
# ホームディレクトリに clone（バージョン 1.5.1 を指定）
git clone -b 1.5.1 https://github.com/raspberrypi/pico-sdk.git \
    --recurse-submodules ~/pico-sdk
```

> サブモジュール（TinyUSB 等）を `--recurse-submodules` で同時に取得します。後から取得する場合は `cd ~/pico-sdk && git submodule update --init --recursive` を実行してください。

### 3.2 環境変数の設定

```sh
# 現在のシェルセッションのみ有効
export PICO_SDK_PATH=$HOME/pico-sdk
```

永続化するには、使用するシェルの設定ファイルに追記します。

| シェル | 設定ファイル | 追記内容 |
|--------|------------|---------|
| zsh | `~/.zshrc` | `export PICO_SDK_PATH=$HOME/pico-sdk` |
| bash | `~/.bashrc` | `export PICO_SDK_PATH=$HOME/pico-sdk` |
| fish | `~/.config/fish/config.fish` | `set -x PICO_SDK_PATH $HOME/pico-sdk` |

追記後、設定ファイルを再読み込みします（例: `source ~/.zshrc`）。

## §4 ファームウェアのビルド

### 4.1 推奨: ビルドスクリプト経由

プロジェクトに付属の `scripts/build.sh` を使用するのが最も簡単です。CMake 4.x 利用時のワークアラウンドも自動で適用されます。

```sh
# PICO_SDK_PATH が設定済みであることを確認
echo $PICO_SDK_PATH   # 空欄の場合は §3.2 の手順で設定する

# 通常ビルド（configure 済みなら差分のみビルド）
./scripts/build.sh

# クリーンビルド（build/ を削除して最初からビルド）
./scripts/build.sh --clean
```

**ビルド成功時の出力例:**

```
[build.sh] CMake 4.x.y 検出 → CMAKE_POLICY_VERSION_MINIMUM=3.5 を設定
[build.sh] configure 実行
...
[build.sh] build 実行
...
[build.sh] 成功: build/src/kkbd_usb.uf2
```

### 4.2 成果物の確認

ビルドが成功すると `build/src/kkbd_usb.uf2` が生成されます。

```sh
ls -la build/src/kkbd_usb.uf2
# -rw-r--r--  1 user group  XXXXXX  May  5 12:00 build/src/kkbd_usb.uf2
```

ファイルサイズは数十 KB〜数百 KB 程度です。

### 4.3 手動ビルド（スクリプトを使わない場合）

```sh
export PICO_SDK_PATH=$HOME/pico-sdk

# CMake 4.0 以上を使用する場合は以下も必要
export CMAKE_POLICY_VERSION_MINIMUM=3.5

cmake -S . -B build -G Ninja
cmake --build build
```

> **CMake 4.x 利用時の注意**: Pico SDK 1.5.1 に同梱されている TinyUSB のサブビルド（`pioasm` / `elf2uf2`）が古い `cmake_minimum_required` を持つため、CMake 4.0 以上ではビルドエラーが発生します。`CMAKE_POLICY_VERSION_MINIMUM=3.5` を**シェルで `export`** することで子 CMake 呼び出しに環境変数が伝播し、この問題を回避できます（CMakeLists.txt 内の `set(ENV{})` ではビルド時のサブ cmake 起動に伝播しないため不可）。Pico SDK 2.x へのアップグレードで本対応は不要になります。

## §5 Pico への書き込み

### Step 1: BOOTSEL モードで接続

1. Raspberry Pi Pico 上の **BOOTSEL ボタン**（白い小さなボタン）を**押し続けたまま**、USB Micro-B ケーブルで PC に接続する
2. PC のファイルマネージャに **`RPI-RP2`** というボリューム（ドライブ）が表示されることを確認する
3. BOOTSEL ボタンを離す

```
┌──────────────────────────────────────────────────────┐
│  ポイント: BOOTSEL を押しながら USB を差し込む        │
│  ボタンを先に離してから接続すると通常起動になります  │
└──────────────────────────────────────────────────────┘
```

### Step 2: .uf2 ファイルのコピー

OS に応じて以下のコマンドを実行します。

| OS | コマンド / 操作 |
|----|----------------|
| macOS | `cp build/src/kkbd_usb.uf2 /Volumes/RPI-RP2/` |
| Linux | `cp build/src/kkbd_usb.uf2 /media/$USER/RPI-RP2/` |
| Windows（コマンドプロンプト） | `copy build\src\kkbd_usb.uf2 E:\` （ドライブレターは環境による） |
| Windows（エクスプローラ） | `RPI-RP2` ドライブへドラッグ＆ドロップ |

> **注意**: コピー中に USB が切断されたように見えることがありますが、これは正常な動作です。Pico が BOOTSEL モードを終了して自動再起動するためです。

### Step 3: 自動再起動と動作確認

コピー完了後、Pico は自動的に再起動してファームウェアが動作を開始します。

**確認方法:**

| 確認項目 | 期待する動作 |
|---------|------------|
| LED 点灯（常時点灯） | 電源投入直後に点灯（BOOT 状態） |
| LED 低速点滅（500ms 周期） | キーボード未接続の WAIT_DEVICE 状態 |
| シリアル端末での起動バナー | `KKBD-USB v0.1 (Phase 6) - Waiting for USB keyboard...` が UART から出力される |

LED ステータスの詳細は [04_使い方.md § 3 LED ステータスの読み方](04_使い方.md) を参照してください。

## §6 ホスト側ユニットテストの実行（任意）

ファームウェアビルドとは独立して、ホスト PC 上でユニットテスト（キーマップ・設定・キーリピートのロジック検証）を実行できます。Pico 実機がなくてもテスト可能です。

```sh
# テスト用 CMake プロジェクトを configure
cmake -S tests -B build-tests \
    -DKKBD_PHASE4_DONE=ON \
    -DKKBD_PHASE5_DONE=ON \
    -DKKBD_PHASE6_DONE=ON

# ビルド
cmake --build build-tests

# テスト実行
ctest --test-dir build-tests --output-on-failure
```

全テストが `Passed` になれば正常です。テスト戦略の詳細は [`docs/design/テスト戦略.md`](../design/テスト戦略.md) を参照してください。

## §7 トラブルシューティング（簡易）

詳細な対処法は **[07_トラブルシューティング](07_トラブルシューティング.md)** を参照してください。

| 症状 | 考えられる原因 | 対処 |
|------|--------------|------|
| `PICO_SDK_PATH が未設定です` | 環境変数が設定されていない | `export PICO_SDK_PATH=$HOME/pico-sdk` を実行する |
| `PICO_SDK_PATH が存在しません` | SDK が clone されていない、またはパスが間違っている | §3.1 の手順で SDK を clone し、パスを確認する |
| `arm-none-eabi-gcc: command not found` | ARM クロスコンパイラ未インストール | §2 の手順でインストールする |
| `tusb.h` 等のインクルードエラー | Pico SDK のサブモジュール未取得 | `cd ~/pico-sdk && git submodule update --init --recursive` を実行する |
| `Compatibility with CMake < 3.5 has been removed` | CMake 4.x と Pico SDK 1.5.1 の互換性問題 | `scripts/build.sh` を使用する、または `export CMAKE_POLICY_VERSION_MINIMUM=3.5` を実行してから手動ビルドする |
| `RPI-RP2` ボリュームが表示されない | BOOTSEL を押さずに接続 / 充電専用ケーブル使用 | BOOTSEL を押しながら接続したか確認。USB ケーブルがデータ通信対応（充電専用でない）ものを使用する |
| LED がまったく点灯しない | 書き込み失敗 / 電源供給の問題 | 別の USB ポート・ケーブルで再試行。BOOTSEL モードで再接続し、`.uf2` を上書きコピーする |
| ビルドが途中でエラー終了する | SDK バージョン不一致 / 依存ライブラリ不足 | Pico SDK v1.5.1 以上を使用しているか確認。エラーメッセージをもとに不足ライブラリを確認する |
