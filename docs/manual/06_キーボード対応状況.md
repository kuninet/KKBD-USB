## 1. はじめに

本章では「自分の USB キーボードが KKBD-USB で使えるか？」を判断するための情報をまとめています。

KKBD-USB は USB HID（Human Interface Device）仕様に準拠したキーボードクラスのデバイスに対応しています。一般的な USB 有線キーボードの大多数は対応範囲内に含まれますが、キーボードの種類・配列・使用するキーによっては制限があります。

> **対応の基準について: 本ボードは送信専用 I/F です。**
>
> KKBD-USB はキーボードからのキー入力を ASCII コードに変換して UART で SBC へ送信する「単方向送信専用」ボードです。そのため、Caps Lock LED・NumLock LED・Scroll Lock LED などの「ホスト → キーボード方向」の LED 制御機能は使用しません。キーボード側の LED 状態は KKBD-USB では制御されません（Caps Lock の ON/OFF はキーボード自体の内部状態に依存します）。

以下の情報を確認して、お使いのキーボードが対応しているかどうかを判断してください。

## 2. 対応キーボードクラス

### 2.1 対応デバイス

| 種別 | 対応状況 | 備考 |
|------|---------|------|
| USB 有線キーボード（HID Keyboard クラス準拠） | 対応 | 一般的な市販キーボードの大多数が該当 |
| USB ワイヤレスキーボード（USB レシーバー型） | 対応 | キーボード本体は無線だが、PC に刺さるレシーバーが USB HID として動作するもの |
| テンキー付きキーボード | 対応 | テンキーの数字・`Enter`・`.` に対応（§5 参照） |
| 複数 HID インターフェース搭載キーボード | 対応 | メディアキー等の追加インターフェースは無視し、Boot Keyboard インターフェースのみ使用 |

### 2.2 対応の技術的基準

KKBD-USB は TinyUSB の USB ホスト HID クラスドライバを使用し、**USB HID 1.11 仕様の Boot Protocol（Boot Keyboard）**で動作します。

Boot Protocol は最もシンプルな HID キーボードプロトコルで、BIOS 起動時のキーボード操作にも使われる枯れた仕様です。市販の USB キーボードの大多数がこのプロトコルに対応しています。

## 3. 非対応キーボード

| 種別 | 対応状況 | 理由・補足 |
|------|---------|----------|
| Bluetooth キーボード（USB アダプタなし） | **非対応** | KKBD-USB は USB ホストのみ対応。Bluetooth 通信機能は未実装 |
| Bluetooth キーボード（USB レシーバー型） | 条件付き対応 | USB レシーバー（ドングル）が USB HID Keyboard クラスとして動作する場合は対応の可能性あり。ただし動作確認は必要 |
| USB-PS/2 変換アダプタ経由のキーボード | 要確認 | 変換アダプタが USB HID Keyboard として正しくエミュレートしているかによる |
| 独自プロトコルキーボード（ゲーミング専用 SDK 等） | **非対応** | 一部のゲーミングキーボードは専用ドライバが必要な独自プロトコルを使用する場合がある |
| USB Hub 経由の接続 | 未検証 | USB Hub を介した接続は動作保証外。直接接続を推奨 |

> **ゲーミングキーボードについて**: 多くのゲーミングキーボードは USB HID Keyboard クラスにも対応しており、そのまま使用できます。一部のハイエンドモデルで独自プロトコル専用となっているものは動作しません。

## 4. 配列対応状況

### 4.1 US 配列（101/104 キー）

| 状態 | 詳細 |
|------|------|
| **完全対応** | KKBD-USB のキーマップは US 配列を基準に実装されています。US 配列キーボードを使用した場合、キーボードの刻印と送信内容が一致します |

### 4.2 JIS 配列（106/109 キー）

| 状態 | 詳細 |
|------|------|
| **部分対応**（US 配列として動作） | JIS 配列キーボードを接続した場合でも、英数字・制御キー・ASCII 記号は動作します。ただし、HID キーコードを US 配列基準で解釈するため、**キーボードの刻印と送信内容が一致しない場合があります** |

**JIS 配列を使用した場合の具体例**:

| JIS 刻印キー | 期待される文字（JIS 流） | 実際に送られる文字（US 配列基準） |
|------------|----------------------|-------------------------------|
| `@`（P の右） | `@` | `[`（US 配列では同じ位置が `[` キー） |
| `[`（Enter の右） | `[` | `]`（US 配列では同じ位置が `]` キー） |
| Shift+`2` | `"` | `@`（US 配列では Shift+2 が `@`） |
| Shift+`6` | `&` | `^`（US 配列では Shift+6 が `^`） |
| `¥`（バックスラッシュ位置） | `¥` | `\`（US 配列のバックスラッシュキーとして扱われる） |
| 全角/半角 | 全角/半角切り替え | （無視）JIS 固有キーは未対応 |
| 無変換 / 変換 | IME 操作 | （無視）JIS 固有キーは未対応 |
| カタカナ/ひらがな | IME 操作 | （無視）JIS 固有キーは未対応 |

> **JIS 配列の完全対応について**: JIS 配列専用のキーマップ実装は別 Issue として検討中です。現時点では US 配列キーボードの使用を推奨します。

## 5. 対応キー一覧

### 5.1 完全対応キー

| カテゴリ | 対応キー | 送信内容 |
|---------|---------|---------|
| 英小文字 | `a`〜`z` | `a`〜`z`（0x61〜0x7A） |
| 英大文字 | Shift+`a`〜`z` | `A`〜`Z`（0x41〜0x5A） |
| 数字（メイン） | `0`〜`9` | `0`〜`9`（0x30〜0x39） |
| 記号（Shift なし） | `-`, `=`, `[`, `]`, `\`, `;`, `'`, `` ` ``, `,`, `.`, `/` | 対応する ASCII 記号 |
| 記号（Shift あり） | Shift+`1`〜`0` → `!`, `@`, `#`, `$`, `%`, `^`, `&`, `*`, `(`, `)` | 対応する ASCII 記号 |
| 記号（Shift+記号） | Shift+`-`, `=`, `[`, `]`, `\`, `;`, `'`, `` ` ``, `,`, `.`, `/` → `_`, `+`, `{`, `}`, `\|`, `:`, `"`, `~`, `<`, `>`, `?` | 対応する ASCII 記号 |
| Escape | Esc | 0x1B |
| Tab | Tab | 0x09 |
| Space | Space | 0x20 |
| Backspace | Backspace | 0x08（BS） |
| Enter | Enter（メイン） | CR / LF / CRLF（JP1/JP2 ジャンパー設定による） |

### 5.2 制御文字（Ctrl+キー）

| 操作 | 送信内容 | 制御文字名 |
|------|---------|----------|
| Ctrl+A | 0x01 | SOH |
| Ctrl+B | 0x02 | STX |
| Ctrl+C | 0x03 | ETX |
| Ctrl+D | 0x04 | EOT |
| Ctrl+E〜Z | 0x05〜0x1A | （省略） |
| Ctrl+`[` | 0x1B | ESC |
| Ctrl+`\` | 0x1C | FS |
| Ctrl+`]` | 0x1D | GS |
| Ctrl+`-` | 0x1F | US |

Ctrl が押されている場合は Shift より優先されます（Ctrl+Shift+A → 0x01）。

### 5.3 テンキー対応

| テンキー | 送信内容 | 前提条件 |
|---------|---------|---------|
| テンキー `0`〜`9` | `0`〜`9` | NumLock ON を想定（NumLock OFF 時は動作しない場合あり） |
| テンキー `.` | `.` | NumLock ON を想定 |
| テンキー Enter | CR / LF / CRLF | メイン Enter と同じ動作 |

> **NumLock について**: KKBD-USB は NumLock の状態を追跡しません。テンキーが正しく動作しない場合は、キーボード側で NumLock を ON にしてください。

### 5.4 修飾キー

| キー | 動作 |
|------|------|
| 左/右 Shift | 大文字・記号変換に使用（単独押しは無視） |
| 左/右 Ctrl | 制御文字生成に使用（単独押しは無視） |
| 左/右 Alt | **無視**（未対応） |
| GUI（Windows/Cmd）キー | **無視**（未対応） |

### 5.5 未対応キー一覧

以下のキーを押しても UART には何も送信されません（無視されます）。

| カテゴリ | 具体的なキー |
|---------|------------|
| ファンクションキー | F1〜F12 |
| カーソルキー | 上下左右矢印 |
| 編集キー | Insert、Delete、Home、End、Page Up、Page Down |
| システムキー | Print Screen、Scroll Lock、Pause/Break |
| トグルキー | Caps Lock、Num Lock（トグル操作自体は無視） |
| メニューキー | Menu、Application |
| JIS 固有キー | 半角/全角、無変換、変換、カタカナ/ひらがな、¥、ろ |
| 修飾キー（単独） | Shift、Ctrl、Alt、GUI（単独押し） |

## 6. 同時押し対応

KKBD-USB は USB HID Boot Protocol の仕様に準拠し、最大 **6 キー同時押し**に対応します。

| 項目 | 内容 |
|------|------|
| 最大同時押し数 | 6 キー（修飾キー除く） |
| 修飾キー | 修飾キー（Shift/Ctrl）は別途検出されるため、6 キー制限に含まれない |
| 6 キー超過時 | HID 仕様の ErrorRollOver（0x01）が報告され、KKBD-USB は超過分のキー入力を無視します |

> **実用上**: 通常のタイピングでは同時押しが 6 キーを超えることはほとんどありません。ゲームで多数のキーを同時押しするような用途には不向きです（そもそも KKBD-USB はゲーム用途を想定していません）。

## 7. 既知の制限事項

| 制限事項 | 詳細 | 回避策・予定 |
|---------|------|------------|
| **キーマップが US 配列固定** | JIS 配列キーボードを接続すると、刻印と送信内容が一致しないキーがある | US 配列キーボードの使用を推奨。JIS 対応は別 Issue で検討中 |
| **キーリピート速度が固定** | 初回遅延 500ms、リピート間隔 50ms は変更できない（再ビルドが必要） | ソースコードの定数（`KEYREPEAT_INITIAL_DELAY_MS`、`KEYREPEAT_INTERVAL_MS`）を変更してリビルドすることで調整可能 |
| **NumLock 状態追跡なし** | テンキーの NumLock ON/OFF 状態を KKBD-USB は管理しない | キーボード側で NumLock を ON にして使用する |
| **Enter キーはリピートしない** | Enter キーを長押しても、行末コードは 1 回だけ送信される | 仕様（意図的な動作）。誤送信防止のため |
| **Alt/GUI キー未対応** | Alt キーや Windows キーは現在無視される | 将来バージョンで対応検討 |
| **Caps Lock の状態反映なし** | キーボード側の Caps Lock 状態を KKBD-USB は検知しない（キーボードが HID レポートで送るキーコードに従う） | Caps Lock を OFF にした状態で使用することを推奨。大文字は Shift+英字で入力する |
| **F1〜F12・矢印キー等が未対応** | カーソル制御・ファンクションキーは UART に送信されない | 対象 SBC（SBC6800 等）がこれらのキーを必要としない運用を前提としています |
